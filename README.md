# Генератор бюлетенів REEFMC

Веб-сторінка для формування щомісячних бюлетенів REEFMC (пожежі на території
України) у форматі .docx. Працює повністю у браузері — без сервера, без
бекенду, без збереження даних десь у хмарі. Готова для розміщення на
**GitHub Pages**.

## Що вміє

- Форма для номера бюлетеня, дати випуску та періоду звітності.
- Завантаження двох карт (зображень) прямо з комп'ютера — вставляються
  у документ автоматично.
- Дві редаговані таблиці (як в оригінальному бюлетені): статистика пожеж по
  Україні та статистика пожеж у природоохоронних територіях (Emerald
  Network), кожна — з окремою підтаблицею "зона бойових дій".
- Можливість додавати / видаляти рядки (області) — кількість рядків не
  фіксована.
- Автоматичний підрахунок підсумкових рядків ("Total, ha") та відсотка
  впливу зони бойових дій.
- Кнопка "Сформувати Word-файл" — генерує .docx та одразу пропонує
  завантажити.

## Швидкий старт (розміщення на GitHub Pages)

1. Створіть новий репозиторій на GitHub (наприклад `reefmc-bulletin`).
2. Завантажте туди всі файли з цієї теки (`index.html`, `style.css`,
   `app.js`, `assets/logo.png`, цей `README.md`) — зберігаючи структуру
   тек.
3. У репозиторії відкрийте **Settings → Pages**.
4. У розділі **Build and deployment** оберіть джерело **Deploy from a
   branch**, гілку `main`, теку `/ (root)`, натисніть **Save**.
5. Через 1–2 хвилини сторінка стане доступна за адресою на кшталт:
   `https://<ваш-логін>.github.io/reefmc-bulletin/`

Після цього просто відкривайте це посилання щомісяця, заповнюйте нові дані
і качайте готовий Word-файл.

## Як користуватись

1. Впишіть номер бюлетеня, дату випуску та період (від / до).
2. Завантажте дві карти (ті самі PNG/JPG, які ви раніше вручну вставляли
   у Word).
3. У таблицях 1 і 2 відредагуйте цифри по областях. Назви областей вже
   підставлені за зразком поточного бюлетеня — просто змінюйте значення.
   Якщо потрібно прибрати область (немає пожеж) — натисніть "✕" біля
   рядка. Якщо з'явилась нова — "+ Додати область".
4. Натисніть **«Сформувати Word-файл (.docx)»** — файл одразу
   завантажиться на ваш комп'ютер, готовий до фінального форматування
   у Word (за потреби — підправити розриви сторінок, шрифти заголовків
   тощо).

## Важливі обмеження цієї версії

- **Немає прямої інтеграції з Google Диском** (за вашим вибором) — фото
  завантажуються вручну через звичайний файловий діалог браузера.
- Верстка документа побудована **програмно** (через бібліотеку `docx`),
  а не є точною XML-копією вашого оригінального файлу. Це дає надійність
  і легкість підтримки, але деякі дрібні деталі форматування (наприклад,
  дворядкова "зшита" шапка таблиці з об'єднаними клітинками) спрощено до
  одного рядка заголовків. Розміри сторінки, шрифт (Times New Roman),
  межі таблиць та загальна структура розділів відтворені відповідно до
  оригіналу.
- Для роботи сторінки потрібне інтернет-з'єднання в браузері (бібліотека
  `docx` підвантажується з CDN `esm.sh`). Якщо хочете повністю офлайн-
  версію — можна завантажити файл бібліотеки в репозиторій і підключати
  локально (скажіть — підготую такий варіант).
- Пояснювальні примітки та розділ "3. Methodology" — статичний текст,
  узятий з поточного шаблону (він не змінюється місяць до місяця). Якщо
  колись зміниться формулювання методології — треба поправити текст
  безпосередньо в `app.js` (константи `METHODOLOGY_PARAGRAPHS` та
  `EXPLANATORY_NOTE_*`).

## Структура файлів

```
reefmc-bulletin/
├── index.html      — форма введення даних
├── style.css        — стилі сторінки
├── app.js           — логіка: рендер таблиць + генерація .docx
├── assets/
│   └── logo.png      — логотип REEFMC (сталий, з оригінального шаблону)
└── README.md
```

## Подальші можливі покращення

- Пряма інтеграція з Google Drive Picker (потребує створення Google
  Cloud проєкту та Client ID) — щоб обирати карти прямо з Диска без
  ручного завантаження.
- Збереження введених даних між сесіями (наприклад, у localStorage), щоб
  не вводити все заново, якщо сторінка перезавантажилась.
- Точніша XML-копія оригінального макета (дворядкова шапка таблиці з
  об'єднаними клітинками, розриви сторінок 1-в-1 як у зразку).

Якщо будь-що з цього актуально — дайте знати, доробимо.
